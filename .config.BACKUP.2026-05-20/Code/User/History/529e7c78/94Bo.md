# Spec Navegador WebView

## 1. Propósito

Componente de navegación web integrada. Permite visualizar y manipular páginas web dentro de la suite, con vinculación directa al código fuente mediante Source Maps reversos.

## 2. Responsabilidades

- Renderizar páginas web (WebView2/WebKit)
- Navegación URL, back, forward, reload
- DevTools integrado para debugging
- Source Maps reversos (DOM → código fuente)
- Interceptar requests (para testing)
- Gestión de cookies/sesiones aisladas

## 3. Estados UI

| Estado       | Descripción       |
| :----------- | :---------------- |
| **EMPTY**    | Sin URL ingresada |
| **LOADING**  | Cargando página   |
| **READY**    | Página cargada    |
| **ERROR**    | Error de carga    |
| **DEVTOOLS** | DevTools abierto  |

## 5. Acciones del Usuario

| Acción           | Input              | Resultado            |
| :--------------- | :----------------- | :------------------- |
| **Navegar**      | Enter en URL bar   | Carga URL            |
| **Back**         | Click ⟲ o Alt+←    | Página anterior      |
| **Forward**      | Click ⟳ o Alt+→    | Página siguiente     |
| **Reload**       | Click ⟳ o Ctrl+R   | Recarga página       |
| **DevTools**     | Click F12 o ⋮      | Abre DevTools        |
| **Inspeccionar** | Click en elemento  | Highlight + DOM path |
| **View Source**  | Right-click → View | Muestra source raw   |

## 6. Source Maps Reverso

**Propósito:** given un elemento DOM, encontrar el archivo fuente que lo generó.

### 6.1 Flujo

```
1. Usuario hace click en elemento del DOM
     ↓
2. DevTools devuelve DOM path: body > div#app > section.hero
     ↓
3. Consultar Source Map inverso:
    DOM path → archivo fuente + línea
     ↓
4. Abrir Editor en archivo correcto, línea highlighted
```

### 6.2 Formato Source Map

```json
{
  "version": 3,
  "sources": ["src/App.tsx", "src/components/Hero.tsx"],
  "mappings": [
    { "dom": "body>div#app>section.hero", "src": "src/components/Hero.tsx:15" }
  ]
}
```

## 7. Integración con Backend

### 7.1 IPC Commands

```
// Rust → Frontend
"web:loaded"         → Página terminó de cargar
"web:error"         → Error de carga
"web:dom:click"     → Click en elemento DOM

// Frontend → Rust
"web:navigate"      → Navega a URL
"web:back"         → Página anterior
"web:forward"       → Página siguiente
"web:reload"       → Recarga
"web:script:inject" → Inyecta script para testing
```

## 8. Integración con Headless Guardian

| Input         | Acción                                         |
| :------------ | :--------------------------------------------- |
| URL cargada   | Notificar a HG para validar scripts inyectados |
| Request hecho | HG puede interceptar/\_blockear requests       |
| Form submit   | HG valida datos antes de enviar                |
| Cookie set    | HG puede validar policies                      |

## 9. Casos Edge

| Escenario            | Comportamiento               |
| :------------------- | :--------------------------- |
| CORS blocked         | Mostrar error claro          |
| Certificado inválido | Warning + opción de proceder |
| Redirect loop        | Detectar y bloquear after N  |
| Memory alto por tab  | Warning, sugerir cerrar      |
| Offline              | Cache local si disponible    |
